## What I Practiced

### Day 22 – Git Basics
- Installed and configured Git
- Initialized first repository
- Understood the `.git` folder
- Practiced staging, committing, and viewing history
- Learned the Working Directory → Staging Area → Repository flow

### Day 23 – Branching & GitHub
- Created and switched branches
- Understood `HEAD` and branch pointers
- Connected local repo to GitHub (remote)
- Practiced `git push`, `git pull`, `git fetch`
- Learned Clone vs Fork and how to keep a fork updated

### Day 24 – Advanced Git Workflows
- Fast-forward merge vs merge commit
- Intentionally created and resolved merge conflicts
- Rebased a feature branch onto main
- Squash merge to clean up commit history
- Used `git stash` to handle work interruptions
- Cherry-picked specific commits across branches

### Day 25 – Reset, Revert & Branching Strategies
- Practiced `git reset --soft`, `--mixed`, `--hard`
- Used `git revert` to safely undo pushed commits
- Compared reset vs revert for local vs shared branches
- Studied GitFlow, GitHub Flow, and Trunk-Based Development
- Used `git reflog` as a recovery safety net

### Day 26 – GitHub CLI (`gh`)
- Installed and authenticated `gh`
- Created and managed repos from terminal
- Created issues, pull requests, and releases via CLI
- Monitored GitHub Actions workflow runs
- Explored `gh api`, gists, aliases, and automation use cases
